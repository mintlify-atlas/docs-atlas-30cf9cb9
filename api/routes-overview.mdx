The `ZonawebWebsite` controller registers six HTTP routes. Five are standard website page routes that render Odoo QWeb templates; one is a JSON-RPC API endpoint for programmatic status checks.

<Info>
All routes use `auth='public'`, so no login is required to access them. The five `type='http'` routes also set `website=True`, enabling Odoo's website context (layout, multi-website support, etc.).
</Info>

## Route summary

| Method | Path | Type | Auth | CSRF | Description |
|--------|------|------|------|------|-------------|
| GET | `/servicios` | http | public | — | Renders the services page |
| GET | `/portafolio` | http | public | — | Renders the portfolio page |
| GET | `/nosotros` | http | public | — | Renders the about us page |
| GET | `/licencias` | http | public | — | Renders the bidding request form |
| POST | `/submit-bidding` | http | public | required | Submits a bidding request |
| POST | `/api/bidding-status` | json | public | — | Returns the status of a bidding request |

## Page routes

The first four routes are read-only pages that accept no meaningful query parameters. They render a fixed QWeb template and return an HTML response.

## Authentication and CSRF

All routes use `auth='public'`. This means:

- Requests from unauthenticated visitors are accepted.
- Odoo's public user is used for database operations that don't call `sudo()`.
- Both controllers that write data call `sudo()` explicitly.

The `/submit-bidding` route sets `csrf=True` (the default for `type='http'` POST routes). Any HTML form that POSTs to this endpoint must include the Odoo CSRF token in a hidden field named `csrf_token`.

<Warning>
The `/api/bidding-status` route uses `type='json'`, which follows the JSON-RPC 2.0 envelope format. It does **not** use the standard HTML form CSRF mechanism.
</Warning>
