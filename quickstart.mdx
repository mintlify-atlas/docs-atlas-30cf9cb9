This guide walks you through everything needed to have a working corporate website running immediately after [installing the module](/installation).

---

<Steps>
  <Step title="Activate the website module and visit the homepage">
    1. Log in to your Odoo 18 backend.
    2. Navigate to **Website** from the main app menu.
    3. If this is the first time, the Website configurator will open. Complete the brief setup wizard (or skip it) to reach the website editor.
    4. Click **Visit Website** (top-left of the editor toolbar) to open the public-facing homepage.

    The homepage includes a hero carousel with three slides (`hero1.jpg`, `hero2.jpg`, `hero3.jpg`). The carousel uses CSS class `.zw-carousel-slide` and dot navigation with `.zw-carousel-dot`.

    <Tip>
      The homepage hero carousel auto-advances every **4500 ms**. Visitors can also jump to any slide by clicking the dot buttons rendered below the carousel.
    </Tip>
  </Step>

  <Step title="Verify all four published pages">
    Open each URL in a new browser tab (or in an incognito window to test as a public visitor):

    | Page | URL | What you should see |
    |---|---|---|
    | Servicios | `/servicios` | Services overview with service cards |
    | Portafolio | `/portafolio` | Portfolio gallery with category filter buttons |
    | Nosotros | `/nosotros` | About page with team section and animated counters |
    | Solicitar Licitación | `/licencias` | Bidding request form |

    All four pages are published and indexed out of the box — no manual publishing step is required.

    <Note>
    If a page returns a 404, the most common cause is that the Odoo server was not restarted after module installation. Restart the server and try again.
    </Note>
  </Step>

  <Step title="Test the bidding form at /licencias">
    Navigate to `/licencias`. The form collects the following fields:

    **Required fields** (the form will not submit without these):

    | Field | Label | Type |
    |---|---|---|
    | `company_name` | Nombre de la Empresa | Text |
    | `contact_name` | Nombre de Contacto | Text |
    | `email` | Correo Electrónico | Text |
    | `service_type` | Tipo de Servicio | Select (`software`, `infrastructure`, `support`, `security`, `consulting`, `other`) |
    | `project_description` | Descripción del Proyecto | Textarea |

    **Optional fields:**

    | Field | Label | Type |
    |---|---|---|
    | `phone` | Teléfono | Text |
    | `estimated_budget` | Presupuesto Estimado | Number |
    | `estimated_timeline` | Cronograma Estimado | Select (`normal`, `urgent`, `flexible`) |
    | `company_sector` | Sector de la Empresa | Select (`retail`, `manufacturing`, `services`, `technology`, `healthcare`, `education`, `finance`, `other`) |
    | `specific_requirements` | Requerimientos Específicos | Textarea |

    Fill in all required fields and click the submit button. The form posts to `/submit-bidding` (CSRF-protected, POST only).

    **After submission:**
    - A new `zonaweb.bidding.request` record is created in the database with state `draft` and an auto-generated reference number (e.g., `ZW-0001`).
    - A confirmation email is sent to the provided address if the `zonaweb_website.email_template_bidding_confirmation` mail template is configured.
    - The visitor is redirected to a thank-you page that displays the generated bidding reference ID.

    <Note>
    If the email cannot be sent (e.g., no outgoing mail server configured), the system logs a warning and the bidding request is still saved. The form submission does not fail silently.
    </Note>
  </Step>

  <Step title="Check submitted requests in the Odoo backend">
    After a test submission, verify the record was created:

    1. Go to the Odoo backend.
    2. Navigate to **Sales → Licitaciones → Solicitudes**.
    3. The submitted request appears in the list view with columns: Reference, Empresa, Contacto, Email, Tipo de Servicio, Estado, and Fecha.
    4. Click the record to open the form view. You will see the full request details plus the chatter (message thread) at the bottom, powered by `mail.thread`.

    From the form view you can move the request through its lifecycle using the status bar buttons:

    | Action | Button | Visible when |
    |---|---|---|
    | Confirmar | Primary | State is `draft` |
    | Aprobar | Success | State is `confirmed` |
    | Rechazar | Danger | State is `draft` or `confirmed` |

    The kanban view (accessible from the view switcher) groups records by state with color-coded badges: green for approved, yellow for confirmed, and red for rejected.
  </Step>

  <Step title="Customize the navigation menu">
    The module installs four top-level menu entries automatically:

    | Menu label | URL | Sequence |
    |---|---|---|
    | Servicios | `/servicios` | 10 |
    | Portafolio | `/portafolio` | 20 |
    | Nosotros | `/nosotros` | 30 |
    | Licitaciones | `/licencias` | 40 |

    To customize the navigation:

    1. From the public website, click **Edit** in the website editor toolbar.
    2. Open the **Pages** menu (or click directly on the navigation bar) and select **Edit Menu**.
    3. Reorder, rename, add, or remove menu items using the drag-and-drop editor.
    4. Click **Save** to apply the changes.

    Alternatively, manage menus programmatically via **Website → Configuration → Menus** in the backend.

    <Warning>
    Renaming or removing a generated menu item in the editor does not delete the underlying `website.page` or its route. The URL will still respond even if the menu link is removed.
    </Warning>
  </Step>
</Steps>
