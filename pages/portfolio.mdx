## JS filter behavior

The filter is implemented in `zonaweb_script.js`:

```javascript
const filterBtns = document.querySelectorAll('.zw-filter-btn');
const pfCards    = document.querySelectorAll('.zw-pf-card[data-cat]');

filterBtns.forEach(btn => {
    btn.addEventListener('click', () => {
        filterBtns.forEach(b => b.classList.remove('active'));
        btn.classList.add('active');
        const cat = btn.dataset.filter;
        pfCards.forEach(card => {
            const show = cat === 'all' || card.dataset.cat === cat;
            card.style.display = show ? 'block' : 'none';
        });
    });
});
```

**How it works:**

<Steps>
  <Step title="Click a filter button">
    The clicked button receives the `active` class; all others lose it.
  </Step>
  <Step title="Read the filter value">
    The `data-filter` attribute of the clicked button is read as `cat`.
  </Step>
  <Step title="Show or hide cards">
    Each `.zw-pf-card[data-cat]` is shown (`display: block`) if `cat === 'all'` or if the card's `data-cat` matches `cat`. Non-matching cards are set to `display: none`.
  </Step>
</Steps>

<Note>
The filter is purely client-side. All six cards are always present in the DOM; visibility is toggled with `display`.
</Note>

## Scroll reveal

All `.zw-pf-card` elements are also wired to the scroll reveal observer defined in `zonaweb_script.js`. Cards start at `opacity: 0` and `translateY(24px)`, then animate in with a staggered delay of `i * 0.08s` as they enter the viewport.
