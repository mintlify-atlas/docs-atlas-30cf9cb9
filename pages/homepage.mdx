The homepage extends Odoo's built-in `website.homepage` template through the `zonaweb_homepage` template key. It replaces the default `#wrap` content with six full-width sections.

## Hero section

The hero is split into a two-column grid: content on the left and an image carousel on the right.

### Content column

| Element | Value |
|---|---|
| Label | "Soluciones Tecnológicas Profesionales" |
| Heading | "Transformamos tu empresa con tecnología de vanguardia" |
| Subheading | Brief summary of the four service areas |
| Primary CTA | "Solicitar propuesta" → `/licencias` |
| Secondary CTA | "Ver servicios" → `/servicios` |

### Animated stats

Three counters animate from 0 to their target value when scrolled into view (see [JS behaviors](#js-behaviors)):

| `data-count` | Suffix | Label |
|---|---|---|
| `10` | `+` | Años de experiencia |
| `120` | `+` | Proyectos completados |
| `98` | `%` | Clientes satisfechos |

### Image carousel

Three slides cycle automatically. Each slide contains a single `<img>` tag:

| Slide | File | Alt text |
|---|---|---|
| 1 (active) | `hero1.jpg` | Desarrollo de software |
| 2 | `hero2.jpg` | Infraestructura |

Dot navigation buttons (`zw-carousel-dot`) let visitors jump to any slide manually.

## JS behaviors

All interactive behaviors are initialized by `zonaweb_script.js` on `DOMContentLoaded`.

### Carousel auto-play

```javascript
setInterval(() => goTo(current + 1), 4500);
```

The active slide advances every **4500 ms**. Clicking a dot calls `goTo(i)` directly. The `active` class is toggled on both the slide (`zw-carousel-slide`) and the corresponding dot (`zw-carousel-dot`).

### Animated counters

Elements with a `data-count` attribute are observed with `IntersectionObserver` (threshold `0.5`). When 50 % of the element is visible, a `setInterval` increments the displayed number every 16 ms until it reaches the target:

```javascript
const step = Math.ceil(end / (duration / 16)); // duration = 1800 ms
```

### Scroll reveal animations

Cards (`zw-service-card`, `zw-portfolio-card`, `zw-plan-card`) start invisible and shifted 24 px downward. An `IntersectionObserver` (threshold `0.1`) sets `opacity: 1` and `transform: translateY(0)` when each card enters the viewport. Stagger delay increases by `0.08s` per card index.

### Navbar shadow

When `window.scrollY > 20` a `box-shadow` is applied to the main navbar; it is removed when the user scrolls back to the top.
